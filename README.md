# Grand Line Coder

A gamified coding interview preparation platform themed around One Piece. Choose your path — conquer algorithms as a Pirate or master system design as a Marine.

## Paths

**Pirates (DSA)** — Fight through the Grand Line solving algorithmic challenges. Your bounty rises with every problem conquered.

**Marines (LLD/HLD)** — Maintain world order through system design mastery. Climb the ranks from Ensign to Fleet Admiral.

## Rank Progression

Each rank requires **100 points** to advance.

| Action | Points |
|--------|--------|
| First correct submission | +25 |
| Wrong submission | -5 |
| Skip question | -2 |
| Use hint | -5 |

### Pirate Ranks
Rookie → Super Rookie → Supernova → Yonko Commander → Yonko → Pirate King → **GOL D. ROGER**

### Marine Ranks
Ensign → Lieutenant → Captain → Rear Admiral → Vice Admiral → Admiral → Fleet Admiral → **MONKEY D. GARP**

Supreme ranks (GOL D. ROGER / MONKEY D. GARP) require more than just points — you must meet excellence criteria like first-try success rate, category completion, and streaks.

## Tech Stack

### Backend
- **Python 3.11+** / **FastAPI** / **Pydantic v2**
- **MongoDB** (Motor async driver) — primary database
- **Redis** — sessions & caching
- **Judge0** — sandboxed code execution for DSA problems
- **Claude / OpenAI** — system design question generation & evaluation

### Frontend (planned)
- React 18 / Vite / TailwindCSS / Monaco Editor

## Hybrid Question System

- **DSA (Pirate)** — Questions are picked from a curated problem bank with verified test cases
- **System Design (Marine)** — Questions are dynamically generated by an LLM, tailored to the user's weak areas. Falls back to curated problems if the LLM is unavailable

## Getting Started

### Prerequisites
- Python 3.11+
- MongoDB
- Redis
- API keys for OAuth (Google/GitHub) and AI services (OpenAI/Anthropic)

### Quick Start

```bash
cd server
cp .env.example .env
# Fill in your API keys in .env
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### With Docker

```bash
cd server
cp .env.example .env
docker-compose up
```

The API will be available at `http://localhost:8000`.

### Verify

- Health check: `http://localhost:8000/health`
- API docs: `http://localhost:8000/docs`

### Run Tests

```bash
cd server
pytest tests/unit/ -v
```

## API Endpoints

### Authentication
```
GET  /api/v1/auth/google            # Google OAuth login
GET  /api/v1/auth/github            # GitHub OAuth login
GET  /api/v1/auth/me                # Get current user
POST /api/v1/auth/refresh           # Refresh JWT
```

### Problems
```
GET  /api/v1/problems/next          # Get next problem (AI-adjusted difficulty)
GET  /api/v1/problems/random        # Get random problem
GET  /api/v1/problems/{id}          # Get specific problem
POST /api/v1/problems/{id}/skip     # Skip problem (-2 pts)
```

### Submissions
```
POST /api/v1/submissions            # Submit code
GET  /api/v1/submissions/{id}       # Get result
POST /api/v1/submissions/run        # Run code (no scoring)
GET  /api/v1/submissions/{id}/status # Poll status
```

### AI Services
```
POST /api/v1/ai/hint                # Get hint (-5 pts)
POST /api/v1/ai/explain             # Solution explanation
POST /api/v1/ai/evaluate            # Evaluate system design
```
