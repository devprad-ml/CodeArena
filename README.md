# CodeArena

A gamified coding interview preparation platform. Choose your path — conquer algorithms as a **Fighter** or master system design as a **Sentinel**.

---

## How It Works

1. **Sign in** with Google or GitHub
2. **Choose your path** — Fighter (DSA) or Sentinel (System Design)
3. **Solve problems** in a timed coding environment
4. **Earn points** and climb the ranks
5. **Get AI-powered feedback** — hints, explanations, and adaptive difficulty

---

## Dual Path System

| Path | Focus | Content | Progression |
|------|-------|---------|-------------|
| **Fighter** | Arena Competitor | DSA (Arrays, Trees, Graphs, DP, etc.) | Rating increases |
| **Sentinel** | System Defender | LLD/HLD (System Design, OOP, Architecture) | Rank promotion |

### Hybrid Question System

- **Fighter (DSA)** — Curated problem bank with verified test cases. Questions are picked based on your rank and AI-adjusted difficulty.
- **Sentinel (System Design)** — Dynamically generated by an LLM, tailored to your weak areas. Falls back to curated problems if the LLM is unavailable.

## Scoring

| Action | Points |
|--------|--------|
| First correct submission | +25 |
| Wrong submission | -5 |
| Skip question | -2 |
| Use hint | -5 |

**Example:** Correct after 2 wrong attempts = +25 - 10 = **+15 points**

Minimum 5 points for any correct submission, regardless of wrong attempts.

Each rank requires **100 points** to advance.

| Rank | Rating | Points | Difficulty |
|------|--------|--------|------------|
| Novice | 0 | 0 | Easy |
| Brawler | 100 | 100 | Easy-Medium |
| Striker | 300 | 200 | Medium |
| Gladiator | 1000 | 300 | Medium-Hard |
| Champion | 3000 | 400 | Hard |
| Grandmaster | 5000 | 500 | Hard-Expert |
| **LEGEND** | ∞ | 600 | Expert |

### Sentinel Ranks (System Design)

| Rank | Title | Points | Difficulty |
|------|-------|--------|------------|
| Guardian | Defender | 0 | Easy |
| Warden | Strategist | 100 | Medium |
| Protector | Architect | 200 | Medium-Hard |
| Sovereign | Master Builder | 300 | Hard |
| Architect Supreme | Supreme Commander | 400 | Hard-Expert |
| **ORACLE** | The Visionary | 500 | Expert |

### Supreme Ranks

Supreme ranks (LEGEND / ORACLE) **cannot be achieved by grinding alone**. You must meet all qualification criteria:

**LEGEND (Fighter, 600 pts):**
- 80% first-try success rate
- 10+ expert problems solved
- All DSA categories completed
- 7-day active streak

**ORACLE (Sentinel, 500 pts):**
- 75% first-try success rate
- 85% average AI evaluation score
- 10+ LLD and 10+ HLD problems completed
- 5+ perfect scores (95%+)

If you have enough points but don't qualify, you stay at the previous rank with a progress tracker showing what's left.

## Features

### Coding Environment
- **Monaco Editor** (VS Code engine) with syntax highlighting
- **7 languages** — Python, JavaScript, Java, C++, C, Go, Rust
- **Configurable timer** — 15 / 30 / 45 / 60 minutes
- **Sandboxed execution** via Judge0 with test case validation
- **Real-time results** — pass/fail per test case, runtime, memory usage

### AI-Powered Features
- **Adaptive difficulty** — RAG-based system adjusts question difficulty to your performance
- **Dynamic question generation** — LLM generates system design questions targeting your weak areas
- **Hints** — 3 levels (subtle → moderate → detailed), costs -5 points each
- **Solution explanations** — post-submission breakdown with complexity analysis
- **AI evaluation** — system design answers scored by AI with per-criterion feedback

### User Profile & Stats
- Dual-path progress tracking (independent Fighter and Sentinel progression)
- Submission history with code, results, and points
- Achievement badges
- Streak tracking (current and best)
- Category performance breakdown

## Architecture

```
Client (React) ──► FastAPI ──► MongoDB / Redis
                      │
                      ├──► Judge0 (code execution)
                      └──► Claude / OpenAI (AI services)
```

The backend follows a **layered architecture**:

```
API Endpoints → Services → Repositories → Database
                   ↓
            External APIs (Judge0, Claude, OpenAI)
```

| Layer | Directory | Responsibility |
|-------|-----------|---------------|
| API | `app/api/` | HTTP handling, validation, routing |
| Services | `app/services/` | Business logic, orchestration |
| Repositories | `app/db/repositories/` | Database CRUD operations |
| Models | `app/models/` | MongoDB document schemas |
| Schemas | `app/schemas/` | Request/response shapes |
| Core | `app/core/` | Auth, security, exceptions |

## API Endpoints

### Authentication
```
GET  /api/v1/auth/google              # Google OAuth login
GET  /api/v1/auth/google/callback     # Google OAuth callback
GET  /api/v1/auth/github              # GitHub OAuth login
GET  /api/v1/auth/github/callback     # GitHub OAuth callback
GET  /api/v1/auth/me                  # Get current user
POST /api/v1/auth/logout              # Logout
POST /api/v1/auth/refresh             # Refresh JWT
```

### Users
```
GET  /api/v1/users/profile            # Get user profile
PUT  /api/v1/users/profile            # Update profile
GET  /api/v1/users/stats              # Get user statistics
GET  /api/v1/users/submissions        # Submission history
GET  /api/v1/users/achievements       # Achievement list
PUT  /api/v1/users/preferences        # Update preferences
```

### Problems
```
GET  /api/v1/problems/next            # Next problem (AI-adjusted difficulty)
GET  /api/v1/problems/random          # Random problem with filters
GET  /api/v1/problems/{id}            # Specific problem
GET  /api/v1/problems/categories      # List categories for a path
POST /api/v1/problems/{id}/skip       # Skip problem (-2 pts)
```

### Submissions
```
POST /api/v1/submissions              # Submit code for evaluation
GET  /api/v1/submissions/{id}         # Get submission result
POST /api/v1/submissions/run          # Run code without scoring
GET  /api/v1/submissions/{id}/status  # Poll submission status
```

### AI Services
```
POST /api/v1/ai/hint                  # Get hint (-5 pts)
POST /api/v1/ai/explain               # Solution explanation
POST /api/v1/ai/evaluate              # Evaluate system design answer
```

### WebSocket (planned)
```
WS   /ws/submission/{submission_id}   # Real-time submission updates
```

## Getting Started

### Prerequisites
- Python 3.11+
- MongoDB
- Redis
- API keys: Google & GitHub OAuth, OpenAI or Anthropic, Judge0

### Quick Start

```bash
cd server
cp .env.example .env
# Fill in your API keys in .env
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### With Docker

```bash
cd server
cp .env.example .env
docker-compose up
```

The API will be available at `http://localhost:8000`.

### Verify

- Health check: `http://localhost:8000/health`
- Swagger docs: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

### Run Tests

```bash
cd server
pytest tests/unit/ -v
```

### Database Setup

```bash
# Create indexes
python scripts/create_indexes.py

# Seed problem data
python data/seed_db.py
```
